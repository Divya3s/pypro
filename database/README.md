# IssueDesk SQLite database

This folder contains the SQLite database assets for the core Python IssueDesk backend.

## Files

- `issuedesk.sqlite` — ignored local SQLite database generated by `app.py` at runtime.
- `schema.sql` — idempotent schema and seed data loaded by the backend at startup.

## Running the backend

From the repository root, run:

```bash
python3 app.py
```

The backend uses only Python's standard library. On startup it creates or updates the ignored runtime database `issuedesk.sqlite` from `schema.sql`, serves `index.html`, and exposes JSON endpoints for reports and comments.

## API endpoints

- `GET /api/reports` — returns the newest reports, with optional `?type=Bug` style filtering.
- `GET /api/reports/{id}` — returns a single report with comments for the detail page.
- `POST /api/reports` — creates a report from the frontend form.
- `PUT /api/reports/{id}` — updates a report from the detail page.
- `DELETE /api/reports/{id}` — deletes a report and its related comments/history.
- `POST /api/comments` — adds a comment to a saved report.


## Download archive

When `app.py` is running, `GET /download/issuedesk-files.zip` returns a ZIP archive containing the project files and the locally generated SQLite database.
